## Purpose
Provide clear definitions of terms used across the documentation.

## Scope
Covers:
- Technical terms
- Project-specific terminology
- Documentation concepts

Does NOT cover:
- General programming knowledge

## Content

### ADR (Architecture Decision Record)
A document capturing an important decision, including context, alternatives, and consequences.

### Proposal
An ADR in **Proposed** state.

### Feature
A functional unit of the system that delivers user or system value.

### Architecture
The high-level structure of the system, including components and their interactions.

### Subsystem
A distinct part of the system with a specific responsibility.

### API (Application Programming Interface)
A defined interface for communication between components or external systems.

### Resource
External or supporting material such as:
- Hardware schematics
- UI designs
- SharePoint documents
- Task tracking links (e.g: Asana)

### Single Source of Truth
A principle where information exists in only one authoritative location.

### PR (Pull Request)
A change submission that should:
- Include code changes
- Validate/update documentation

### Deprecated
Marked as no longer recommended but still present.

### Superseded
Replaced by a newer alternative.
