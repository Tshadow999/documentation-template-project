## Start here

> This is the entry point for the documentation.
> If you are new to this project, read this document first.

## Purpose
This documentation is the **single source of truth** for the project.

It exists to:
- Explain how this documentation system works
- Capture architectural decisions
- Document features and APIs
- Enable fast onboarding

## What you should do first

1. Read this page
2. Go to [[Project Summary]]
3. See [[Table of Contents]]
4. Explore:
   - [[System Overview]]  to understand the system
   - [[ADR Index]] to understand decisions
5. If developing:
   - Go to [[Getting Started]]

### If you are developing
Before coding:
- Check relevant documentation:
  - [[System Overview]]
  - [[ADR Index]]

During development update documentation when:
- You add or change a feature
- You change architecture
- You make a significant decision (create a new ADR)

### If you are reviewing (PRs)
Documentation is part of the PR.
Always ask:
- Is documentation still correct?
- Should something be updated or added?

## Documentation principles

- **Single source of truth**
  - Do not duplicate information

- **Keep it concise**
  - Prefer bullet points over long text

- **Use diagrams where useful**
  - Prefer Excalidraw over draw.io

- **Link, don’t repeat**
  - Reference other documents instead of copying

- **Keep it up to date**
  - Documentation evolves with the system

## Key documents

- [[Project Summary]] to start understanding the project
- [[System Overview]] to view the overall system structure
- [[ADR Index]] to see why decisions were made
- [[Getting Started]] to setup your development environment

## Expectations

As a developer, you are expected to:
- Read relevant documentation before coding
- Keep documentation up to date
- Create ADRs for:
  - Architectural decisions
  - Major features
  - Tech stack changes
