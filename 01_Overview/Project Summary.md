## Purpose
Provide a standardized, low-friction documentation structure that ensures consistency, clarity, and maintainability across the entire project lifecycle.

## Scope
Covers:
- Documentation structure and conventions
- How different types of documentation are organized
- How developers interact with documentation

Does NOT cover:
- Implementation details of the actual product
- Tool-specific tutorials (unless in Development section)

## Content
This project defines a **documentation-first approach**:
- Documentation is created **before and during development**
- Each change is reflected through:
  - Feature documentation updates
  - ADRs (for significant decisions)

Core principles:
- Single source of truth
- Minimal duplication
- Low maintenance overhead
- High readability (bullet points, diagrams, no walls of text)

The structure is optimized for:
- Fast onboarding
- Clear architectural understanding
- Traceability of decisions
- Scalable documentation as the project grows
