## Purpose
Identify all parties that interact with or depend on the documentation.

## Scope
Covers:
- Roles interacting with documentation
- Their expectations and needs

Does NOT cover:
- Organizational hierarchy
- Responsibilities outside documentation

## Content

### Developers
- Primary users and contributors
- Needs:
  - Fast understanding of system
  - Clear feature and architecture docs
  - Minimal friction for updates

### New Team Members
- Onboarding audience
- Needs:
  - High-level overview
  - Glossary
  - Clear entry points

### External Stakeholders (Optional)
- Clients, partners, auditors
- Needs:
  - Read-only understanding
  - Clear API documentation
  - Decision traceability (ADR)
